# UI System Rules

## Visual direction
- The product should feel like a game for builders, not a SaaS admin panel.
- Use a polished, modern, high-energy interface with strong hierarchy and clear rewards.
- Keep the UI cohesive across dashboard, generator, project workspace, planner, and export flows.

## Experience principles
- Mobile first.
- Rich empty states, loading states, progress states, and reward feedback.
- Use motion tastefully for delight and momentum, not noise.
- Prefer cards, panels, map-like planners, quest logs, streak indicators, progress rings, and achievement moments where appropriate.

## Interaction expectations
- Dialogs, popovers, tabs, drawers, sheets, accordions, and tooltips should work.
- Toasts and status indicators should communicate progress and completion clearly.
- Provide mock content and realistic placeholders so screens feel alive before backend logic exists.

## Avoid
- Generic dashboard aesthetics
- Overly flat, lifeless enterprise layouts
- Placeholder lorem ipsum everywhere
- Dense screens without prioritization
